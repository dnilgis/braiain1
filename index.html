METHODOLOGY: THE COGNITIVE GAUNTLET

1. Heavy Context Ingestion

To simulate a production RAG environment, we inject a dense ~2,500 character narrative ("The Story of Ouroboros") into the context window. This measures the model's 'Cold Start' capability—its ability to rapidly ingest, tokenize, and comprehend new data before generating a single token of output.

2. Strict Schema Compliance

We do not accept free-form text. The model is tasked with extracting specific entities from the injected context and formatting them into a strict JSON schema. This tests the model's instruction-following precision and its utility for structured data extraction tasks.

3. Symbolic Logic Verification

We challenge the model with the 'Bat & Ball' problem—a classic test of cognitive reflection. This effectively separates models that simply predict probable tokens (guessing $0.10) from those that perform multi-step logical reasoning to arrive at the correct conclusion ($0.05).

4. Syntactic Code Generation

The final gauntlet requires generating a recursive Python function for a Fibonacci sequence, complete with type hints and docstrings. We use Regex to validate that the output is not just textually coherent, but syntactically executable code.

5. Composite Scoring Algorithm

The final 'Braiain Score' is not arbitrary. It is a weighted algorithmic composite: 50% based on the Quality Score (accuracy on tasks), 30% on Total Speed (normalized against a 30s timeout), and 20% on Responsiveness (TTFT + TPS). This penalizes models that are smart but slow, or fast but hallucinate.

DISCLAIMERS

  • Context Load: Scores reflect performance under load (RAG simulation), not empty prompts.
  • Variability: Results depend on US-East server conditions and cloud provider region status at the time of testing.
  • Independence: 100% Open Source project. No financial affiliations with any provider.
  • Evolving Standard: Methodologies are subject to refinement as AI capabilities advance.
